# 본 프로젝트의 목적
1. 해당 프로젝트는 프로토타입으로써 Must-Have 구현만을 목표로 한다.
2. 해당 서비스는 모바일(ios, android)로 배포된다.
    - ios: TestFlight
    - android: Google Play internal testing 
3. 해당 서비스는 익명 고민 공유 플랫폼으로써 동작한다.
4. 해당 프로젝트는 한국어만을 지원한다.

# Must-Have: 필수 핵심 기능
- 고민을 작성하고 사용자 정보(성별, 관심 분야) 기반으로 LLM이 판단하기에 적절한 고민 답변자 3명에게 전송한다. LLM은 반드시 적절한 사용자 3명을 찾아내야만 하며 최적 3명이 없을 경우 차선으로라도 찾아야 한다.
- 앱 실행 시 첫 화면이 내 답변을 기다리는 고민들 목록
- 앱 첫 실행 → 성별, 관심 분야 선택
- LLM 필터링 기능을 통해 부적절한 고민 게시, 고민 답변을 차단한다.
- 고민에 대한 답변을 받은 후 좋아요 버튼 및 후기 작성
- 예제 질문 데이터를 포함하여 앱 배포 초기 사용자가 없는 상황에서도 미리 준비된 예제 질문 데이터에 대해 사용자가 답변할 수 있도록 한다. 예제 질문 데이터는 실제 사용자가 질문한 것처럼 보이게 한다.

# Won't-Have: 이번 프로젝트 구현에서는 전혀 고려하지 않는다.
- 게임적인 기능 (랭킹 등)
- 초기 접속 → AI 대화를 통한 사용자 파악
- 이전의 고민들을 기록하는 보관소
- 앱 내에서 캡처방지기능
- 인증받은 사람들만 답변할 수 있도록하는 기능
- 고민에 대한 답변을 받은 후 1~5 스케일의 평점 기능(좋아요 대체) 및 후기 작성
- 고민 답변 평점이 임계점(프로젝트에서 임의 상수 설정) 이하인 경우 해당 유저에게 고민 답변 요청이 적게 가도록 함.
- 고민 답변을 초기에는 3명에게 요청한다. 1시간이 지나도 답변이 전송되지 않는 경우 추가로 3명에게 요청한다. 또 1시간이 지나도 전송되지 않으면 추가로 3명... (반복)
- 관심 분야 카테고리 선택 후 세부 카테고리가 떠서 더 자세한 관심 분야 선택 가능
- 고민 게시 관심 분야, 고민 답변 관심 분야 분리
